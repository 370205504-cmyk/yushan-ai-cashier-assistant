## 二、系统要求

### 2.1 硬件要求

```
最低配置：
- CPU：Intel i3 / AMD Ryzen 3 及以上
- 内存：4GB RAM
- 硬盘：500MB 可用空间
- 显示器：分辨率 1280x720 及以上

推荐配置：
- CPU：Intel i5 / AMD Ryzen 5 及以上
- 内存：8GB RAM
- 硬盘：1GB 可用空间
```

### 2.2 软件要求

```
操作系统：Windows 10/11 64位
Node.js：18.x LTS 或更高版本
浏览器：Chrome/Firefox/Edge 最新版
网络：局域网连接（用于多设备同步）
```

### 2.3 Node.js 安装

如果您的电脑还没有安装 Node.js，请按以下步骤操作：

1. 访问 Node.js 官网：https://nodejs.org/zh-cn/
2. 下载 **18.x LTS** 版本（推荐）
3. 运行安装程序，一路点击"下一步"即可
4. 安装完成后，打开 CMD 或 PowerShell，输入：
   ```
   node --version
   ```
   如果显示版本号（如 v18.x.x），说明安装成功

## 三、安装部署

### 3.1 下载安装包

从 GitHub 下载最新版本：
```
https://github.com/370205504-cmyk/yushan-ai-cashier-assistant/releases
```

下载文件：`yushan-ai-cashier-assistant-v5.0.0-windows.zip`

### 3.2 解压安装

1. 将压缩包解压到 **D:\YushanAI** 目录
   > 注意：路径中不要有中文、空格或特殊字符

2. 解压后的目录结构：
   ```
   D:\YushanAI\
   ├── lambda\              # 程序目录
   │   ├── database\        # 数据库脚本
   │   ├── services\       # 服务模块
   │   ├── routes\          # 路由接口
   │   └── server.js        # 服务入口
   ├── data\                # 数据存储目录
   ├── backups\             # 备份目录
   ├── logs\                # 日志目录
   ├── 启动服务.bat         # 启动脚本
   ├── 停止服务.bat         # 停止脚本
   ├── 初始化数据库.bat     # 初始化数据库
   ├── 使用说明.txt         # 使用说明
   └── .env                 # 配置文件
   ```

### 3.3 首次启动

1. 双击 **启动服务.bat**
2. 如果是首次运行，会自动安装依赖（需要1-3分钟）
3. 安装完成后会自动初始化SQLite数据库
4. 服务启动后，会显示：
   ```
   ===========================================
   服务启动中，请勿关闭此窗口...
   
   打开浏览器访问: http://localhost:3000
   ===========================================
   ```

### 3.4 访问系统

打开浏览器（推荐 Chrome），访问：

| 地址 | 说明 |
|------|------|
| http://localhost:3000 | 顾客端（扫码点餐界面） |
| http://localhost:3000/admin | 管理端（后台管理） |
| http://localhost:3000/setup | 配置向导（首次设置） |

## 六、配置说明

### 6.1 配置文件

编辑 `.env` 文件修改配置：

```bash
# 服务端口（默认3000）
PORT=3000

# 监听地址（默认0.0.0.0）
HOST=0.0.0.0

# 环境模式
NODE_ENV=development

# Session密钥（生产环境请修改）
SESSION_SECRET=your-secret-key-here
```

### 6.2 端口修改

如果3000端口被占用：

1. 打开 `.env` 文件
2. 修改 `PORT=3001`（或其他端口）
3. 重启服务

### 6.3 多电脑访问

如果需要其他设备访问（同一局域网）：

1. 查看服务器电脑的IP地址（如 192.168.1.100）
2. 其他设备浏览器访问：`http://192.168.1.100:3000`

## 七、数据管理

### 7.1 数据存储位置

```
lambda/data/cashier.db    # SQLite数据库文件
backups/                   # 自动备份目录
logs/                      # 日志文件
```

### 7.2 手动备份

备份数据只需复制数据库文件：

1. 停止服务（运行 停止服务.bat）
2. 复制 `lambda/data/cashier.db` 到备份位置
3. 重启服务

### 7.3 数据恢复

1. 停止服务
2. 将备份的 `cashier.db` 文件覆盖 `lambda/data/cashier.db`
3. 重启服务

### 7.4 重置数据库

如果需要清空所有数据重新开始：

1. 运行 停止服务.bat 停止服务
2. 删除 `lambda/data/cashier.db` 文件
3. 运行 初始化数据库.bat 重新初始化
4. 运行 启动服务.bat 启动服务

## 八、常见问题

### Q1: 提示"未检测到 Node.js"？

**解决方法**：
1. 确认已安装 Node.js 18+
2. 重启电脑
3. 或重新安装 Node.js

### Q2: 端口被占用？

**解决方法**：
1. 打开 `.env` 文件
2. 修改 `PORT=3001`
3. 重启服务

### Q3: 忘记管理员密码？

**解决方法**：
1. 停止服务
2. 删除 `lambda/data/cashier.db`
3. 运行 初始化数据库.bat
4. 启动服务（账号恢复为 admin/admin123）

### Q4: 依赖安装失败？

**解决方法**：
1. 检查网络连接
2. 清理 npm 缓存：`npm cache clean --force`
3. 重新运行 启动服务.bat

### Q5: 数据库初始化失败？

**解决方法**：
1. 确保路径无中文和空格
2. 确保有写入权限
3. 手动创建 `lambda/data` 目录
