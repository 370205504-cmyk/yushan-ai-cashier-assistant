## 🔴 缺陷1：数据层设计灾难（JSON→数据库）

### 原缺陷描述
- **问题位置**: `lambda/data/*.json` + 所有Service层
- **技术原理**: `fs.writeFile` 覆盖式写入，无锁、无事务、无原子性
- **丢单率**: 5个浏览器并发下单 → 100%丢单

### ✅ 修复验证

#### 1. JSON 文件已替换为 MySQL 数据库
**不再使用 JSON 存储！**

| 检查项 | 验证结果 |
|--------|----------|
| 数据存储 | ✅MySQL + Redis |
| 事务支持 | ✅有（`db.transaction()`） |
| 并发安全 | ✅有（数据库连接池+锁） |

## 🔴 缺陷3：任意价格下单漏洞 → ✅已修复

### 原缺陷描述
- **问题位置**: `orderServiceV2.js`
- **攻击方式**: 传入 0.01 元点 88 元菜品
- **影响**: 直接经济损失

### ✅ 修复验证

#### 1. 双重价格防护
**代码位置**: [routes/order.js#L31-L46](file:///workspace/lambda/routes/order.js#L31-L46)

```javascript

  return res.status(400).json({ success: false, message: '非法请求参数' });

...
```

#### 2. 强制从数据库读取真实价格
**代码位置**: [routes/order.js#L48-L67](file:///workspace/lambda/routes/order.js#L48-L67)

```javascript
// ✅从数据库查询菜品，获取真实价格
const dishes = await db.query(
  'SELECT * FROM dishes WHERE id = ? AND is_available = 1',
  [item.dishId]
);
if (dishes.length === 0) {
  return res.status(400).json({ success: false, message: `菜品不存在: ${item.dishId}` });
}
const dish = dishes[0];

// ✅使用数据库价格计算，完全忽略前端
const quantity = parseInt(item.quantity);
const subtotal = dish.price * quantity;  // dish.price 来自数据库！
totalAmount += subtotal;
```
